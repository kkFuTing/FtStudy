## 面试冲刺 14 天计划（结合现简历技术栈）

### 总体说明

- 面向 Android 高级/负责人岗位，突出机器人通信、Cordova 平台、工程化治理与 Java/Kotlin 双栈能力。
- 每天投入 2.5~3 小时（可拆分为早中晚 3 段），包含学习、输出、演练与复盘。
- 所有输出（STAR 稿、错题集、总结卡、图示）统一存于同一目录，便于随时翻阅。
- 每日结束时，记录“今日收获 + 易错点 + 翌日待办”三行小结。

---

### 第 1 天：核心经历梳理

- **STAR 草稿**：完成 `优必选通信/状态机`、`明源云 Cordova 平台`、`南亿智能家居` 三段 STAR 描述（≤300 字）。
- **数据速查表**：整理崩溃率、插件数量、并发连接、展会支持等关键数字。
- **语音自测**：录制 2 分钟自我介绍，突出 7 年经验、最新项目、技术标签。
- **资料管理**：创建“错题集”“协程原理解构”“Java 速查”三个笔记文档。

### 第 2 天：项目抽象与亮点提炼

- **架构草图**：为三段核心经历绘制架构/流程图，标出关键模块与职责。
- **亮点列表**：每段经历整理 3 条高光点（难点、指标、跨团队成果）。
- **开放题练习**：回答“项目难点是什么”“你的角色”“业务价值体现”等问题。
- **反问准备**：写出 5 个反问（聚焦机器人业务、技术挑战、团队协作）。

---

### 第 3 天：协程基础原理 + Java 并发开篇

- **学习重点**
  - 协程与线程区别，`suspend` → 状态机转换，`Continuation` 概念。
  - Java 内存模型（JMM）：可见性、有序性；`volatile`、`synchronized`、`Lock` 差异。
- **实践**
  - 反编译一个简单 `suspend` 函数，手绘状态机。
  - 编写协程 demo，打印线程名验证挂起与恢复。
  - 写出双重检查锁 DCL 需要 `volatile` 的原因。
- **输出**
  - 完成“协程本质 = 对象”总结卡。
  - 记录 5 道“协程基础”＋2 道“JMM”练习题及答案。

### 第 4 天：调度器机制 + Java 线程池

- **学习重点**
  - `Dispatcher` 实现：Default/IO/Main、`CoroutineDispatcher.dispatch`。
  - `launch` vs `async` vs `runBlocking`、`CoroutineStart.LAZY` 行为。
  - Java 线程池：参数含义、拒绝策略、任务调度流程。
- **实践**
  - 编写多种启动方式的协程 demo，对比执行顺序。
  - 回顾线程池在项目中的使用，整理配置与监控手段。
- **输出**
  - 调度器速查卡（场景 → 选择理由）。
  - 线程池问答 3 题（含实际案例，如插件构建使用线程池）。

### 第 5 天：结构化并发 + JVM/GC

- **学习重点**
  - `Job` 树、`SupervisorJob`、取消传播、异常处理。
  - JVM 类加载机制、双亲委派；Android 特殊性（多 Dex、Instant Run）。
  - GC 策略：分代、CMS/GSS，如何影响 UI 卡顿。
- **实践**
  - demo：父协程取消、子协程异常的不同表现。
  - 总结崩溃率治理中与 GC/内存相关的经验。
- **输出**
  - “协程取消流程图”。
  - JVM 问答 3 题（类加载、GC、Heap 参数）。

### 第 6 天：Flow/Channel + 集合并发

- **学习重点**
  - 冷流/热流、背压模式；`StateFlow`、`SharedFlow`、`channelFlow`。
  - Java 集合底层：`ArrayList`、`HashMap`、`ConcurrentHashMap`。
- **实践**
  - Flow demo：`buffer`、`conflate`、`collectLatest` 行为对比。
  - 讲解 HashMap 扩容、并发风险；ConcurrentHashMap 的 CAS 分段锁。
- **输出**
  - Flow 选择指南表格。
  - 集合底层问答 4 题（含扩容、线程安全、性能差异）。

---

### 第 7 天：网络通信深化

- **内容**
  - WebSocket 心跳、断线重连、消息路由；Proto 兼容策略；HTTP/TCP 基础。
  - Java NIO 与阻塞 IO 对比，结合机器人通信选型。
- **输出**
  - 记录优必选通信的关键参数（心跳间隔、重连策略、并发量）。
  - 练习问答：为什么选 Proto？如何兼容旧客户端？弱网优化手段？

### 第 8 天：状态机与机器人业务

- **内容**
  - 运行模式列表、优先级、互斥规则、异常回退。
  - 多传感器协作、MQTT 告警、现场问题处理。
- **输出**
  - 状态机图示（含安全模式、回退策略）。
  - 现场案例 STAR 描述（LEAP 延迟、潮州部署应急）。

### 第 9 天：Cordova 平台与跨端

- **内容**
  - Cordova 原理、插件通信、生命周期、依赖矩阵治理。
  - Node.js 脚本在预检、自动化发布中的作用；与 Flutter、原生策略对比。
- **输出**
  - 崩溃率降至 0.25‰ 的专项步骤详解。
  - 构建矩阵提升 70% 的具体流程表。

### 第 10 天：音视频与多媒体

- **内容**
  - 直播/互动架构、码率自适应、QoS 方案。
  - 提示音系统优先级、AudioFocus、队列抢占策略。
- **输出**
  - 音视频链路图（互动端、观众端、信令/媒体）。
  - 问答：弱网下如何保障体验？提示音延迟如何优化？

---

### 第 11 天：工程化与稳定性

- **内容**
  - 构建依赖矩阵、预检脚本、CI/CD、隐私合规流程。
  - LeakCanary、Crashlytics、ANR/OOM 治理方法。
  - Java 在工程化中的应用（Gradle 插件、脚本、诊断工具）。
- **输出**
  - “质量治理案例”文档（行动步骤、指标改善、团队协作）。
  - 准备问答：如何处理构建失败？隐私审核流程？如何追踪崩溃指标？

### 第 12 天：行为题深挖

- **内容**
  - 整理 10 个行为题：冲突处理、失败经验、跨团队沟通、压力与成长。
  - 将 Java/Android 技术结合着讲（如推动代码规范、处理遗留代码）。
- **输出**
  - 每题 STAR 答案，控制 1 分钟内表达。
  - 模拟录音，调整语气与节奏。

---

### 第 13 天：全流程模拟

- **流程**
  - 1. 自我介绍（2 分钟）
  - 2. 协程原理问答（10 分钟）
  - 3. 项目深挖（机器人通信、Cordova 平台各 10 分钟）
  - 4. 行为题（5 分钟）
  - 5. 反问（3 分钟）
- **复盘**
  - 记录卡壳点、语句不顺处、数据记忆漏洞。
  - 更新错题集与补充笔记。

### 第 14 天：查漏补缺与心态调整

- **复习清单**
  - 协程原理解构卡、Java 速查表、项目数据速查表、行为题答案。
  - 用白板再画一次：协程状态机、Cordova 架构、状态机优先级。
- **轻量练习**
  - 随机抽 10 道协程/Java 题自问自答。
  - 确认面试当天设备、网络、环境；准备面试后记录模板。

### 面试期日常维护

- **每日 20 分钟**：协程卡片（10 分钟）＋ Java 速查（5 分钟）＋ 项目数据回顾（5 分钟）。
- **面试前 10 分钟**：快速默念自我介绍 + 当天要强调的亮点。
- **面试后**：记录题目、感受、需要补强的点，当晚或次日补齐知识盲区。
